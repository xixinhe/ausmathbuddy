# ADLC — 文档状态流转（Draft → Review → Approved → Active → Deprecated）

所有 `docs/` 下的规划类文档（PRD 等）必须在文件头部标注当前状态，并按以下流程单向流转。

## 状态定义

1. **Draft** — 起草中，内容可能还在变化，尚未拉人评审。
2. **Review** — 已提交评审，等待反馈/修改意见。
3. **Approved** — 评审通过，内容冻结，可以开始按此执行。
4. **Active** — 正在按此文档执行/已上线，是当前生效版本。
5. **Deprecated** — 已被新版本取代或场景失效，仅作历史存档，不再指导开发。

## 规则

- 每个文档头部必须有 `> **Status**: <状态>` 一行，紧跟一句指向本规则的引用。
- 状态只能按 Draft → Review → Approved → Active → Deprecated 单向流转，不允许跳过（例如 Draft 不能直接变 Active）。
- 例外：内容被推翻重做时，可以从任意状态直接打回 **Draft**，但要在文档里简要记一句原因。
- 状态变更本身就是一次值得记录的事件：谁在什么时间因为什么原因把状态往前推进，简单记一行在文档末尾的变更记录里（不需要长篇说明）。
- 一个文档同一时间只有一个状态，不用状态数组或多状态并存。

## 使用方式

- 新建 PRD/设计文档：默认 `Draft`。
- 提交给他人看意见：改成 `Review`。
- 对方确认没问题：改成 `Approved`。
- 开始按此实施或已经上线生效：改成 `Active`。
- 被新版本取代、或需求作废：改成 `Deprecated`，不要直接删除旧文档。

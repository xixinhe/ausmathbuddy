# PRD 超长自动拆分

`docs/` 下任意 PRD 文件超过 1000 行时，视为触发信号，必须拆分成多个 PRD 模块文件，不能继续在单文件里累加内容。

## 触发条件

- 检查方式：`wc -l docs/xxx-prd.md`，超过 1000 行即触发。
- 不用等用户提醒，agent 每次要在某个 PRD 文件里新增大段内容前，先检查行数，预计会超过 1000 行就先拆分，再写入。

## 拆分方式

- 按 PRD 现有的功能模块/章节拆（比如按"目标范围""页面流程""数据""模块拆解"这类一级章节，或按业务功能域），不要按行数硬切。
- 拆出的每个子文件命名为 `docs/<原名>-<模块名>.md`，例如 `docs/prd.md` 拆成：
  - `docs/prd-overview.md`（目标/范围/验收，作为主入口）
  - `docs/prd-<功能模块>.md`（按功能拆的具体模块）
- 保留一个主入口文件（overview），里面用列表链接到所有拆出的子模块，方便找齐全貌。
- 每个拆出的子文件独立标注 [[adlc]] 状态（Draft/Review/Approved/Active/Deprecated），状态不必和主文件一致——不同模块的成熟度可以不同。
- 拆分后，子文件同样遵守"不超过 1000 行"，如果某个子模块自己又长到 1000 行，继续往下拆。

## 原则

- 拆分只按内容边界来，不因为拆分而改动原有内容的措辞/结论（避免拆分变成夹带修改）。
- 拆完之后在主入口文件里留一句简短说明：什么时候因为超过 1000 行拆分的，拆成了哪些文件（呼应 [[adlc]] 的变更记录习惯）。

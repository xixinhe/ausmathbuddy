# ALMO — Always Leave Me Optimized

每次接触代码时，让它比你发现时更好（哪怕只是一点点），而不是原样离开或变得更糟。

## 规则

- 顺手清理明显的死代码、无用的 import、过时的注释——但不做与当前任务无关的大范围重构（避免和 KISS 冲突，改动范围仍需克制）。
- 修复 bug 时，如果发现命名有歧义、逻辑重复，可以顺手小范围优化，但不要借机夹带无关功能。
- 提交前自查：这段代码有没有可以更清晰、更高效但改动很小的地方？
- 不引入新的技术债：不要因为赶时间而写明知有问题的临时代码却不留痕迹。
- 如果暂时没时间做更好的实现，至少留一句清晰的说明（放 PR 描述或 issue，而不是代码注释里堆砌）。

## 边界

- "优化"不等于"重写"。不做超出当前任务范围的架构调整。
- 不为了优化而牺牲可读性（不要写晦涩的一行流代码）。
- 与 [[kiss]] 冲突时，优先 KISS：简单、可读始终优先于炫技式优化。
